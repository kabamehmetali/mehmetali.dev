Rose Flower Delivery App

Rose Flower Delivery App is a capstone project collaboratively developed by Mehmmet Ali, Miguel Angel Gutierrez, Parisam Karimi, and Pang. This app simplifies the process of ordering and delivering flowers efficiently and reliably.

Features

- Real-Time Order Tracking with status updates
- Driver Assignment & Management for efficient deliveries
- Google Maps integration for optimized routing
- Secure Authentication with Firebase
- Persistent Cart and real-time Firestore updates

Getting Started

1. Clone the repository: git clone https://github.com/kabamehmetali/CapstoneProject.git
2. Set up Firebase (Authentication & Firestore) and add google-services.json.
3. Obtain a Google Maps API key and insert it in google_maps_api.xml.
4. Open the project in Android Studio, sync Gradle, and run it on a device.

Capstone Project

The Rose – Flower Delivery App is a capstone project designed to connect local florists with customers through an intuitive digital platform. It is developed collaboratively by a team of students to address the growing need for convenient flower ordering and delivery services in Canada.

Project Summary

In summary, the project delivers a native Android solution that streamlines the process of ordering and delivering flowers. With separate interfaces for customers, store owners, and delivery drivers, the app supports secure payments, real-time order tracking, and efficient order management – ensuring a seamless and reliable experience for all users.

Project Vision

The vision behind the project is to revolutionize the floral retail experience by enhancing customer convenience and supporting local businesses. The app aims to provide a user-friendly interface that improves the online shopping process, boosts local sales for florists, and creates opportunities for delivery personnel, thereby fostering a community-driven ecosystem.

Project/Business Requirements

The business requirements include the development of dedicated Android applications for customers, drivers, and store owners. Core functional requirements involve a real-time order assignment system, secure payment integration, and a responsive web component. Additional non-functional requirements focus on usability, security, and performance to ensure a seamless and reliable service.

Project Plan

The project plan spans from September 16, 2024 to March 28, 2025, divided into planning, development, testing, and deployment phases. Key milestones include the development of customer, driver, and store owner apps; integration of a secure payment system; and final system testing. Regular status meetings, bi-weekly progress reports, and a detailed Gantt chart ensure that deadlines are met and risks are managed.

Requirements Analysis and Design

During the analysis phase, requirements were gathered for distinct user roles with specific functionalities: ordering and payment for customers, inventory and order management for stores, and real-time delivery tracking for drivers. The design phase established a three-app ecosystem using Android and a centralized Firebase backend, ensuring efficient data synchronization, secure authentication, and smooth integration with third-party services.

Wireframes/Mockups

Wireframes and mockups were created using Figma to visualize the user interface for each module. These prototypes focused on a clean, intuitive layout and user-centric navigation, guiding the development process. Early feedback allowed the team to iterate and refine the design for an optimal user experience. the files are located inside documents given above.

Status Reports

Two formal status reports were produced. The first (January 17, 2025) noted some schedule adjustments due to initial delays while maintaining a “green” project health. The second report (April 17, 2025) confirmed that the project was on track after rescheduling key milestones, with final tasks focused on feature enhancement, integration testing, and final deployment preparations.

System Implementation

The implementation uses a native Android approach, built with Android Studio and leveraging Google Firebase as the backend. Features include user authentication, real-time order tracking, and secure payment integration through a third-party API. The system architecture is modular, enabling efficient communication between the customer, driver, and store modules, while thorough testing ensured stability and performance.
